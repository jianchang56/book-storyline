# 长篇质量与状态管理

长篇分批处理、章节超过二十章或人物关系复杂时读取本文件。

## 状态文件

优先使用 `scripts/prepare_book.py` 生成的 `state.json`。每完成一批章节后更新：

- `completed_chapters`：已完成的章节序号；
- `characters`：人物当前姓名、称号、身份和关系；
- `important_items`：重要物品、能力、规则及首次出现章节；
- `unresolved_conflicts`：尚未解决的目标、威胁、谜团和承诺；
- `terminology_notes`：异体字、译名和版本用词；
- `source_uncertainties`：OCR、缺页和章节识别疑点；
- `last_updated`：最后更新时间。

不要把模型推测写入状态文件。无法确认的内容放入 `source_uncertainties`。

## 分批策略

- 每批处理连续章节，不交错处理相距很远的章节。
- 新批次开始前读取上一批状态，而不是只读取上一章梗概。
- 完成一批后检查开头是否承接上一批，结尾是否留下正确的未解决事件。
- 最终合并时按 manifest 顺序，不按文件系统返回顺序。

## 质量指标

逐章检查：

1. 标题与顺序正确。
2. 起因、关键决定、转折和结果完整。
3. 没有提前泄露后文。
4. 每个保留的人名和专名都有情节作用。
5. 没有无作用的风景、名单和重复动作。
6. 梗概能够单独说明本章如何改变故事状态。

全书检查：

1. manifest 中章节全部处理且没有重复。
2. 人物身份、关系和称号变化一致。
3. 重要能力、物品和规则没有提前出现。
4. 未解决冲突最终得到承接，或确实在原书中保持未解决。
5. 同类章节的精简尺度稳定。

## 压缩比例

压缩比例只用于发现异常，不作为硬目标：

- 比相邻章节明显短时，检查是否漏掉事件；
- 比相邻章节明显长时，检查是否残留复述和装饰细节；
- 情节密度确实不同则允许长度不同。
